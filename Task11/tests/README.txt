Инструкция по управлению ручным тестером PyTest

Тестирующая система PyTest позволяет выполнять запуск 
и проверку работы программы для домашнего задания по B-деревьям.

Настройка работы тестера осуществляется посредством указания параметров
в файле "tester.py". 

0) Опционально тестер может пересобрать ваши программы.
Для этого нужно установить флаг remake (по умолчанию равен False) равным True

0.5) Перед запуском тестирования тестер может сгенерировать множество тестовых данных.
Для включения генерации нужно установить флаг generate_tests равным True.
Тестер генерирует файлы с количествами элементов от min_length до max_length (включительно)
с шагом step для каждой пары m из списка m_attempts и k из k_attempts.

Система может работать в двух режимах:
1) массового тестирования, когда запускаются стандартные 
и сгенерированные тесты ('global')
2) тестирования одного теста ('single')

Для выбора режим работы необходимо указать его в переменой mode.

1) Режим массового тестирования.
В этом режиме тестер сначала запускает стандартные тесты, а затем сгенерированные.
Описание стандартных тестов содержится в файле standard_set.py.
Запускаются сгенерированные файлы с количествами элементов от min_length до max_length (включительно)
с шагом step для каждой пары m из списка m_attempts и k из k_attempts.
Запуск происходит с параметрами m и k из списков выше.

2) Режим тестирования одного теста.
В этом режиме нужно указать имя тестируемого файла в переменной file_name
и параметры m и k для запуска.
Опционально можно включить печать дерева. 
Для этого необходимо установить флаг print_tree равным True.
Параметр precision указывает какое количество уровней B-дерева будет напечатано. 

В обоих режимах тестирования возможно включить проверку утечек памяти.
Для этого установить флаг use_valgrind равным True.
Предупреждение: с включенными проверками памяти тестирование происходит медленнее!

Вывод результатов работы осуществляется в терминал и в файл test_log.py.
Если тестер обнаружил у вас ошибку в ответах, то в файле test_log.py
будут выведены Ваш ответ, ответ тестера и для стандартных тестов правильный ответ.
Если вы не хотите осуществлять вывод в терминал установите флаг only_log равным True.
